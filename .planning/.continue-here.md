<decisions_made>

**Milestone v1.3 decisions:**
- Framework-agnostic testing approach — test db.py and jellyfin_library.py directly without Flask coupling
- pytest framework with pytest-mock and responses — industry standard, Python 3.13 compatible
- In-memory SQLite with tmp_path for database testing — complete isolation prevents state leakage
- Mock HTTP requests to external APIs — prevent real Jellyfin/TMDB calls in unit tests
- Basic coverage reporting in Phase 17 — defer advanced coverage thresholds to v1.3+

**Research-backed choices:**
- pytest over unittest — modern fixtures, parametrization, better assertion introspection
- pytest-mock over unittest.mock — cleaner API via mocker fixture, automatic cleanup
- responses over manual mock — standard for mocking requests library calls
- tmp_path over shared test DB — prevents test interference and flaky failures
