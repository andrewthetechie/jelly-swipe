# Plan 23-01: Create Centralized HTTP Client Helper

**Created:** 2026-04-26
**Status:** Planning
**Phase:** 23 — HTTP Client Centralization
**Type:** Implementation

---

## Overview

Create a centralized HTTP client helper function that enforces security best practices across all outbound HTTP requests in the application. This helper will provide consistent timeouts, User-Agent headers, structured logging, and error handling.

---

## Requirements Addressed

- **HTTP-01:** Centralized HTTP Client Helper
- **HTTP-04:** All requests Calls Have Timeouts
- **TEST-01:** HTTP Client Helper Tests

---

## Success Criteria

1. `jellyswipe/http_client.py` exists with `make_http_request()` function
2. Helper function accepts `method`, `url`, `headers`, `params`, `json`, `timeout` parameters
3. Helper sets default User-Agent header: `JellySwipe/1.6 (+https://github.com/andrewthetechie/jelly-swipe)`
4. Helper enforces timeout: default `(5, 30)` for connect/read
5. Helper logs structured outcome: method, url, status_code, duration_ms, success/failure
6. Helper catches requests exceptions and re-raises with context
7. Unit tests validate timeout enforcement, header setting, and error handling
8. All tests pass

## Implementation Plan

### Task 1: Create http_client.py module

**File:** `jellyswipe/http_client.py` (NEW FILE)

**Actions:**
1. Create `jellyswipe/http_client.py` with the following structure:
   - Import required modules: `requests`, `logging`, `time`, `typing`
   - Define `DEFAULT_USER_AGENT` constant
   - Define `DEFAULT_TIMEOUT` constant as `(5, 30)`
   - Implement `make_http_request()` function with full docstring
   - Add structured logging for both success and failure cases
   - Ensure proper exception handling with context preservation

**Function Signature:**
```python
def make_http_request(
    method: str,
    url: str,
    headers: Optional[Dict[str, str]] = None,
    params: Optional[Dict[str, Any]] = None,
    json: Optional[Dict[str, Any]] = None,
    timeout: Tuple[int, int] = DEFAULT_TIMEOUT,
    **kwargs
) -> requests.Response
```

**Key Implementation Details:**
- Add User-Agent header if not present in provided headers
- Use `requests.request()` for flexible HTTP method support
- Log success with: method, url, status_code, duration_ms, success flag
- Log errors with: method, url, duration_ms, error_type, error_message
- Re-raise exceptions with full context
- Measure request duration in milliseconds
- Use `logger.info()` for successful requests
- Use `logger.error()` for failed requests

**Verification:**
- File exists at `jellyswipe/http_client.py`
- Function signature matches specification
- Docstring is complete and accurate
- Logging statements use proper log levels
- Exception handling preserves error context

### Integration Tests

- Test logging integrates with existing logger configuration
- Ensure no conflicts with existing imports

### Edge Cases
- Empty headers dict
- None values for optional parameters
- Very short timeouts
- Network errors (connection refused, timeout)
- HTTP errors (4xx, 5xx)

## Risks and Mitigations

| Risk | Mitigation |
|------|------------|
| Logging configuration conflicts | Use module-level logger, let app configure handlers |
| Timeout values too aggressive | Use reasonable defaults (5s connect, 30s read) |
| Exception handling loses context | Preserve original exception in re-raise |
| Performance overhead from logging | Keep logging lightweight, only log structured data |
