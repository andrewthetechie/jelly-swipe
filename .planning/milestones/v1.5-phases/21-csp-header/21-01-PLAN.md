2. After this line, add the following code:

```python
@app.after_request
def add_csp_header(response):
    """Add Content Security Policy header to block inline scripts and restrict external resources."""
    csp_policy = (
        "default-src 'self'; "
        "script-src 'self'; "
        "object-src 'none'; "
        "img-src 'self' https://image.tmdb.org; "
        "frame-src https://www.youtube.com"
    )
    response.headers['Content-Security-Policy'] = csp_policy
    return response
```

3. Ensure the hook is placed BEFORE any route definitions (the first route is @app.route('/') at line 71)

Critical requirements (per CSP-01, CSP-02, CSP-03):
- MUST use exact CSP policy: `default-src 'self'; script-src 'self'; object-src 'none'; img-src 'self' https://image.tmdb.org; frame-src https://www.youtube.com`
- MUST NOT include 'unsafe-inline' or 'unsafe-eval' in any directive
- MUST apply to all HTTP responses (no route-specific exceptions)
- Policy MUST be set on the response object's headers dictionary

Do NOT add:
- CSP violation reporting (report-uri or report-to)
- Nonce-based CSP
- Any exceptions to the policy
- Conditional CSP logic (policy must be the same for all responses)

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-21-01 | I (Info disclosure) | Content-Security-Policy header | mitigate | CSP policy restricts img-src to 'self' and trusted domain (https://image.tmdb.org), preventing data exfiltration via malicious image URLs |
| T-21-02 | T (Tampering) | Script execution | mitigate | script-src 'self' without 'unsafe-inline' blocks inline scripts and external scripts from untrusted origins |
| T-21-03 | E (Elevation of privilege) | Dynamic code execution | mitigate | Absence of 'unsafe-eval' blocks eval() and similar functions that could execute injected code |
| T-21-04 | S (Spoofing) | Plugin content | mitigate | object-src 'none' blocks all plugins (Flash, Java, etc.) that could be used for XSS or phishing |
