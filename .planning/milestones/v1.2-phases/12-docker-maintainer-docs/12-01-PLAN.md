  <action>Replace the entire Dockerfile with a multi-stage build that:
1. Builder stage:
   - FROM python:3.13-slim as builder
   - WORKDIR /app
   - Install uv via pip: RUN pip install --no-cache-dir uv
   - COPY pyproject.toml uv.lock ./
   - RUN uv sync --frozen --no-install-project (installs dependencies from lockfile, layer caching optimization)
   - COPY . .
   - RUN uv sync --frozen (second sync after source code, installs jellyswipe package)

2. Final stage:
   - FROM python:3.13-slim
   - WORKDIR /app
   - Copy virtual environment from builder: COPY --from=builder /app/.venv /app/.venv
   - Copy package data (templates/static) from builder: COPY --from=builder /app/jellyswipe /app/jellyswipe
   - Create /app/data directory: RUN mkdir -p /app/data
   - EXPOSE 5005
   - Preserve existing CMD: CMD ["gunicorn", "-b", "0.0.0.0:5005", "-k", "gevent", "--worker-connections", "1000", "jellyswipe:app"]

Layer caching optimization: Copy pyproject.toml and uv.lock before source code so dependency layer only rebuilds when dependencies change, not when application code changes. First uv sync uses --no-install-project for maximum caching benefit.

Do NOT add any PyPI publishing steps or workflows. Do NOT use pip install -e . or install from PyPI. Use only the committed uv.lock and uv sync per D-01, D-02, D-03.</action>
