Purpose: The /room/stream SSE endpoint fails with `SystemExit: 1` errors during `time.sleep(POLL)` because Gunicorn's default sync workers are incompatible with long-lived Server-Sent Events connections. Gevent provides coroutine-based concurrency that handles SSE generators correctly.

Output: Docker container using Gunicorn gevent workers, SSE stream works without errors, clients can connect and receive events normally.

# Existing SSE Implementation

From jellyswipe/__init__.py (lines 433-487):
- Route: `/room/stream`
- Generator function with `time.sleep(POLL)` where POLL = 1.5 seconds
- Timeout: 3600 seconds (1 hour)
- Yields SSE events when room state changes
- Current failure: `SystemExit: 1` during sleep in sync workers

# Gap Root Cause

From 11-HUMAN-UAT.md Gap 1:
- Issue: SSE generator fails during `time.sleep(POLL)`
- Root Cause: Gunicorn sync workers block on `time.sleep()`, causing worker timeout
- Test Case: Joining a room hangs, errors, then repeats
- Priority: High — blocking core functionality

# Solution: Gunicorn with gevent workers

Per gap description, option 1 (quickest fix):
- Add gevent to dependencies
- Use `gunicorn -k gevent` worker class
- Gevent provides coroutine-based async I/O compatible with SSE generators
- Minimal code changes (only Dockerfile and dependencies)
</context>

CMD ["gunicorn", "-b", "0.0.0.0:5005", "jellyswipe:app"]

To:

The -k gevent flag sets the worker class to gevent, and --worker-connections 1000 increases the default connection limit for SSE (gevent default is 1000, but explicit is better for SSE use cases).

Reference gap reason: "SSE generator fails with SystemExit: 1 during time.sleep(POLL)" — gevent workers handle generators correctly and don't timeout on sleep operations.
