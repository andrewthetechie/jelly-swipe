## v1.5 Requirements

- [x] **TEST-ROUTE-02**: Add `tests/test_routes_xss.py` with XSS security tests (HTML tag escaping, javascript: URL rejection, script injection prevention) for EPIC-03

- [x] **TEST-ROUTE-03**: Add `tests/test_routes_room.py` with room operation tests (create, join, swipe, match, quit, status, go-solo) covering happy paths and edge cases
  - **Outcome:** 27 tests in Phase 25. All 6 room endpoints covered with full swipe match logic (solo/dual/no match).

- [x] **TEST-ROUTE-04**: Add `tests/test_routes_proxy.py` with proxy route tests (valid/invalid paths, allowlist regex validation, content-type verification) for EPIC-04 SSRF prevention
  - **Outcome:** 16 tests in Phase 26. All `/proxy` branches covered including 8 SSRF attack vectors.

- [x] **TEST-ROUTE-05**: Add `tests/test_routes_sse.py` with SSE streaming tests (event streaming, invalid room handling, state change events, GeneratorExit handling)
  - **Outcome:** 8 tests in Phase 27. SSE state changes, room closure, GeneratorExit, and header verification all tested. Coverage reached 78% for `__init__.py`.

## Out of Scope

| Feature | Reason |
|---------|--------|
| pytest-flask plugin | Flask's built-in test client is sufficient |
| Framework-agnostic route tests | Route testing requires Flask test client |
| Real Jellyfin/TMDB API calls in tests | Unit tests must be isolated |
| Edge case tests (concurrent operations) | Not required for 70% coverage target |
| Performance tests (load testing) | Not required for v1.5 |
| Integration tests (end-to-end workflows) | Separate concern; defer to v2+ |
