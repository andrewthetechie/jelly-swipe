# Plan 01 — Jellyfin server session: login, `/Items` probe, reset/retry

<objective>
Verify and, only if needed, tighten **server-side Jellyfin authentication** so `get_provider()` in `jellyfin` mode obtains a usable access token (API key or username/password per Phase 1 env contract), proves it against **`/Items`**, and recovers from **401** with **`reset()` + single re-auth** parity to Plex (JAUTH-01, JAUTH-02; `03-CONTEXT` D-01–D-07).
</objective>

<threat_model>
| ID | Threat | Severity | Mitigation |
|----|--------|----------|------------|
| T-01 | Token or password in logs/JSON errors | high | Keep generic `RuntimeError` / HTTP messages; never `str(r.text)` from auth responses in user-visible errors |
| T-02 | Unbounded retry loops on bad creds | medium | `_api` retry flag pattern — at most one re-auth after 401 |
</threat_model>
