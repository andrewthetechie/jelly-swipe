# Plan 02 — Migrate `app.py` routes to `get_provider()` / `reset()` (Plex parity, watchlist/auth exception)

<objective>
Remove module-level `_plex_instance`, `_genre_cache`, and free functions `get_plex`, `reset_plex`, `get_plex_genres`, `fetch_plex_movies` from `app.py` (no thin delegating wrappers—clean removal). Route handlers for library flows call `from media_provider import get_provider, reset` instead. `/proxy`, `/plex/server-info`, `/get-trailer/<movie_id>`, `/cast/<movie_id>`, `/genres`, `/movies`, and `create_room` use provider methods. Preserve HTTP **503** when not in Plex mode or Plex URL/token missing for `/proxy`, and **403** for bad/missing `path` rules. **Leave** `/watchlist/add`, `/auth/plex-url`, `/auth/check-returned-pin` as Plex-specific route implementations per CONTEXT D-03; they may call `get_provider().<resolve method>` **only** to deduplicate `fetchItem` retry/reset, documented inline—no new watchlist methods on the ABC. ARC-02: with `MEDIA_PROVIDER=plex`, responses match pre-refactor shapes and status codes.
</objective>

<threat_model>
| ID | Threat | Severity | Mitigation in this plan |
|----|--------|----------|-------------------------|
| T-01 | Open proxy via `/proxy` (arbitrary upstream paths) | high | Route delegates to `get_provider().fetch_library_image` (or chosen method name) **only** after the provider enforces `/library/metadata/` prefix; keep `abort(403)` for bad paths and never pass user input to URL beyond validated path |
| T-02 | Secrets in JSON errors from `str(e)` on provider failures | medium | Keep existing `jsonify({'error': str(e)})` pattern only where already present; ensure provider/`requests` paths do not raise messages containing `X-Plex-Token` or raw tokens; if upstream raises URL-with-token, catch and replace with generic `"Upstream image fetch failed"` for `/proxy` only (document if added) |
| T-03 | Jellyfin partial-route confusion (library routes half-working) | medium | Library routes call `get_provider()` which raises **before** Plex-only logic; no route-level `except` that swallows `RuntimeError` into 200 responses; `/watchlist/add` retains explicit `MEDIA_PROVIDER != "plex"` → 400 JSON guard unchanged |
| T-04 | Regression on `/proxy` status matrix (mode vs env) | medium | Preserve order of checks: `MEDIA_PROVIDER != "plex"` **or** missing `PLEX_URL` / `ADMIN_TOKEN` → `abort(503)`; then missing/bad path → `abort(403)`—match pre-migration `app.py` |
</threat_model>
