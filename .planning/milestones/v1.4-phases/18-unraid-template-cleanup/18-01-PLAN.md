# Plan 01 — Update Unraid template with Jellyfin variables and blank placeholders

<objective>
Replace Plex environment variables with Jellyfin equivalents in both `<Variable>` and `<Config>` sections, and remove fake placeholder values from all masked fields, ensuring users can deploy with a clean template (TEMP-01, TEMP-02, TEMP-03, TEMP-04, UX-01, CONTEXT D-01, D-02).
</objective>

<threat_model>
| ID | Threat | Severity | Mitigation |
|----|--------|----------|------------|
| T-18-01 | Template still references removed Plex variables | high | Verify `grep -v PLEX` succeeds on template; ensure no PLEX_URL or PLEX_TOKEN in file |
| T-18-02 | Fake placeholder values confuse users about blank fields | low | Verify all `<Value>` elements are empty strings and `Default=""` for masked fields |
</threat_model>

<must_haves>
- Template contains `JELLYFIN_URL` and `JELLYFIN_API_KEY` in both `<Variable>` and `<Config>` sections
- Template contains `TMDB_API_KEY` and `FLASK_SECRET` masked variables (already present, just clean placeholders)
- Template does NOT contain `PLEX_URL` or `PLEX_TOKEN` anywhere in the file
- All masked fields (`JELLYFIN_API_KEY`, `TMDB_API_KEY`, `FLASK_SECRET`) have `<Value></Value>` (empty) and `Default=""`
- Non-masked field (`JELLYFIN_URL`) has `<Value></Value>` (empty) and `Default=""`
- No fake placeholder values like "YOUR_PLEX_URL", "YOUR_PLEX_TOKEN", "YOUR_TMDB_API_KEY", "Enter_random_string"
</must_haves>

  <action>
Read unraid_template/jelly-swipe.html, .planning/phases/18-unraid-template-cleanup/18-CONTEXT.md, jellyswipe/__init__.py, and jellyswipe/jellyfin_library.py to understand current state and env var requirements.

In unraid_template/jelly-swipe.html:
1. Remove the <Variable> entries for PLEX_URL and PLEX_TOKEN (lines 40-49 in current template)
2. In the <Variable> section, add these two entries in order after <Environment> opening:
   <Variable>
     <Name>JELLYFIN_URL</Name>
     <Value></Value>
     <Mode/>
   </Variable>
   <Variable>
     <Name>JELLYFIN_API_KEY</Name>
     <Value></Value>
     <Mode>password</Mode>
   </Variable>
3. Keep existing <Variable> entries for TMDB_API_KEY and FLASK_SECRET but ensure their <Value> elements are empty (<Value></Value>) and remove fake placeholders "YOUR_TMDB_API_KEY" and "Enter_random_string"
4. In the <Config> section, remove the two <Config> entries for PLEX_URL and PLEX_TOKEN (lines 69-70 in current template)
5. In the <Config> section, add these two entries in order before TMDB_API_KEY:
   <Config Name="JELLYFIN_URL" Target="JELLYFIN_URL" Default="" Mode="" Description="URL of your Jellyfin server" Type="Variable" Display="always" Required="true" Mask="false"/>
   <Config Name="JELLYFIN_API_KEY" Target="JELLYFIN_API_KEY" Default="" Mode="" Description="Your Jellyfin API key" Type="Variable" Display="always" Required="true" Mask="true"/>
6. Ensure existing <Config> entries for TMDB_API_KEY and FLASK_SECRET have Default="" and no fake placeholder values
7. Update <Overview> section (line 13) to change "Swipe and match movies for Plex or Jellyfin" to "Swipe and match movies for Jellyfin"
8. Update <Description> field (line 28) to change "Swipe and match movies for Plex or Jellyfin" to "Swipe and match movies for Jellyfin"
</action>
