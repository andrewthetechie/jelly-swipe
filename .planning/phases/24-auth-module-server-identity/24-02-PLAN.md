## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → POST/DELETE routes | Untrusted input crosses here; @login_required gate rejects without valid vault entry |
| Client → GET public routes | Open endpoints (genres, server-info, static) — no auth required, no sensitive data returned |
| Client → SSE stream | Cookie-only auth; EventSource cannot send Authorization headers — vault reads from session cookie |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-24-06 | Spoofing | mutation routes | mitigate | @login_required ignores all request headers and body fields; reads identity exclusively from vault via session cookie |
| T-24-07 | Info disclosure | login response | mitigate | Login returns {userId} only; token stored server-side in vault, never in response body (per D-01) |
| T-24-08 | Repudiation | swipes/matches | accept | Ephemeral session data in self-hosted app; no audit trail needed |
| T-24-09 | Elevation of privilege | unauthenticated access | mitigate | All mutation endpoints gated by @login_required; unauthenticated requests get 401 with clear error message |
| T-24-10 | Info disclosure | session cookie | mitigate | HttpOnly (Flask default), Secure=True, SameSite=Lax (per D-13); session_id is opaque, token stays in SQLite |
</threat_model>
