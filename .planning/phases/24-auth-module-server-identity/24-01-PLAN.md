```python
DB_PATH = None  # Set by __init__.py

def get_db():
    """Get a database connection."""
    conn = sqlite3.connect(DB_PATH)
    conn.row_factory = sqlite3.Row
    return conn

def cleanup_expired_tokens():
    """Delete rows from user_tokens older than 24 hours."""
    cutoff = (datetime.now(timezone.utc) - timedelta(hours=24)).isoformat()
    with get_db() as conn:
        conn.execute('DELETE FROM user_tokens WHERE created_at < ?', (cutoff,))
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Flask session cookie → auth.py | Signed cookie provides session_id; tampering prevented by Flask's secret_key signing |
| auth.py → user_tokens SQLite | Trusted internal read/write; no external input reaches SQL without parameterized queries |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-24-01 | Spoofing | login_required | mitigate | Reads identity only from vault (session cookie → user_tokens), never from request headers or body |
| T-24-02 | Tampering | session cookie | mitigate | Flask SecureCookieSessionInterface signs cookies with secret_key — tampering detected and rejected |
| T-24-03 | Info disclosure | create_session | mitigate | Token stored only in user_tokens SQLite, never returned to client or set in JS-accessible session data |
| T-24-04 | Denial of service | create_session | accept | Token cleanup on each login keeps table small; self-hosted app, flood risk is operator's concern |
| T-24-05 | Elevation of privilege | login_required | mitigate | No vault entry = 401; decorator is the single gate for all authenticated endpoints |
</threat_model>
