### `jellyswipe/__init__.py` (utility/shim, import-time)

**Analog:** `jellyswipe/__init__.py` lines 1-8 — the existing `try/except ImportError: pass` guard around `dotenv` is the canonical pattern to copy.

**Existing dotenv guard** (lines 1-8) — the pattern to replicate:
```python
try:
    from pathlib import Path

    from dotenv import load_dotenv

    load_dotenv(Path(__file__).resolve().parent.parent / ".env")
except ImportError:
    pass
```

**Current Flask import block** (lines 10-12) — the block to wrap:
```python
from flask import Flask, send_from_directory, jsonify, request, session, Response, render_template, abort, g
from flask.json.provider import DefaultJSONProvider
from werkzeug.middleware.proxy_fix import ProxyFix
```

**Target Flask import block** (wrap identical to the dotenv guard above):
```python
try:
    from flask import Flask, send_from_directory, jsonify, request, session, Response, render_template, abort, g
    from flask.json.provider import DefaultJSONProvider
    from werkzeug.middleware.proxy_fix import ProxyFix
except ImportError:
    pass
```

**Style rules:**
- No blank line between `try:` and the first import inside the block (mirrors the dotenv guard style).
- `except ImportError:` on its own line, `pass` indented under it — no additional message or logging.
- Do NOT wrap any other lines (gevent guard on line 20 already has its own try/except; leave it unchanged).
- This guard is temporary — Phase 31 replaces the entire `__init__.py` with the FastAPI app factory, at which point the guard disappears naturally (D-02).

### `uv.lock` (config, tool-generated)

**Analog:** `uv.lock` — entirely tool-generated. No code pattern to copy.

**Regeneration command:**
```bash
uv sync
```

**Rules:**
- Run `uv sync` (without `--frozen`) after editing `pyproject.toml`. This resolves the updated dependency graph and rewrites `uv.lock`.
- The Docker build uses `uv sync --frozen` (lines 13 and 19 of the Dockerfile), so the lockfile in the repo must be up-to-date before the Docker build runs.
- Commit the updated `uv.lock` alongside `pyproject.toml` changes — they must stay in sync.
- Do not hand-edit `uv.lock`.
