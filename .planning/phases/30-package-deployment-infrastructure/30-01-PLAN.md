<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| PyPI → local environment | Package resolution fetches packages from public PyPI; no private index |
| Docker build context → image CMD | Dockerfile CMD is the container entrypoint; format determines process model |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-30-01 | Tampering | PyPI package resolution (supply chain) | mitigate | Version floors (`>=`) prevent known-vulnerable versions; `uv.lock` pins exact resolved hashes — lock file is committed and Docker build uses `uv sync --frozen` so the build is reproducible from the committed lockfile |
| T-30-02 | Tampering | Dependency confusion (private vs public namespace) | accept | All package names (fastapi, uvicorn, itsdangerous, etc.) are well-known PyPI names with no private-registry lookalikes; no private index is configured |
| T-30-03 | Elevation of Privilege | Uvicorn `--host 0.0.0.0` exposes port on all interfaces | accept | This is unchanged from the Gunicorn behavior; the container is behind a reverse proxy; `EXPOSE 5005` is documentation only; network isolation is a deploy-time concern outside this phase |
| T-30-04 | Denial of Service | Single-process Uvicorn (no workers) | accept | Single process is intentional (D-05) to avoid SQLite write contention; concurrency is handled by asyncio I/O; this is a known, deliberate tradeoff |
| T-30-05 | Information Disclosure | `try/except ImportError: pass` silences Flask import errors | accept | The guard is transitional (Phase 30 only); it only hides a missing-package ImportError, not logic errors; Phase 31 removes the guard entirely; no sensitive data is suppressed |
</threat_model>
