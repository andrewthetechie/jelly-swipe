<decisions>
## Implementation Decisions

### Test Bridging (Flask import gap between Phase 30 and Phase 31)

- **D-01:** Add `try/except ImportError: pass` guard around the Flask import block in `jellyswipe/__init__.py`. This is a transitional shim — not a logic change. It allows `import jellyswipe.db` and `import jellyswipe.jellyfin_library` to succeed even when Flask is not installed, so framework-agnostic tests (test_db.py, test_jellyfin_library.py, etc.) continue to run.
- **D-02:** The guard is removed naturally in Phase 31 when `jellyswipe/__init__.py` is replaced by the FastAPI app factory. It MUST NOT persist beyond Phase 31.
- **D-03:** Remove `--cov-fail-under=70` from the `[tool.pytest.ini_options]` in `pyproject.toml`. Coverage reporting stays on (`--cov=jellyswipe --cov-report=term-missing`) but no threshold enforced. The threshold is restored in Phase 35 once all 48 tests are migrated and coverage numbers are known.
- **D-04:** Flask-dependent test fixtures (`flask_app`, `client`) will error at fixture setup time — not at import time. This is expected and acceptable for Phase 30. The full test suite is not expected to pass until Phase 35.

### Uvicorn Process Model

- **D-05:** Single-process Uvicorn — no `--workers` flag. Asyncio handles I/O concurrency. Multiple workers would add SQLite write contention risk and are not needed for this migration phase.
- **D-06:** Dockerfile CMD: `CMD ["/app/.venv/bin/uvicorn", "jellyswipe:app", "--host", "0.0.0.0", "--port", "5005"]`
- **D-07:** No `--reload` flag — production image only.

### Package Versions (from DEP-01 and PyPI-verified research)

- **D-08:** Runtime additions: `fastapi>=0.136.1`, `uvicorn[standard]>=0.46.0`, `itsdangerous>=2.2.0`, `jinja2>=3.1.6`, `python-multipart>=0.0.18`
- **D-09:** Dev additions: `httpx>=0.28.1` (TestClient transport for Phase 35)
- **D-10:** Runtime removals: `flask`, `gunicorn`, `gevent`, `werkzeug`
- **D-11:** Keep `requests>=2.33.1` and `python-dotenv>=1.2.2` — these are not Flask-specific and are used for Jellyfin API calls and .env loading respectively
- **D-12:** Do NOT add `pydantic` or `starlette` explicitly — they are pinned as hard transitive deps of fastapi 0.136.1 and explicit pins risk version conflicts

</decisions>
