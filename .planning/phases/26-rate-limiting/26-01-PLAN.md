  <behavior>
    - Test 1: New bucket starts full at capacity — first N consume() calls return True (N = capacity)
    - Test 2: consume() returns False after capacity exhausted
    - Test 3: After waiting sufficient time for refill, consume() returns True again
    - Test 4: retry_after() returns ~0 when tokens available, positive float when exhausted
    - Test 5: Refill rate = capacity/60 tokens per second (e.g., capacity=10 → 0.1667 tokens/sec)
    - Test 6: RateLimiter.check() creates independent buckets per (endpoint, ip) pair
    - Test 7: Hitting limit on endpoint A does NOT affect endpoint B for same IP
    - Test 8: Hitting limit on IP A does NOT affect IP B for same endpoint
    - Test 9: Stale buckets (last access >300 seconds ago) are evicted on next check
    - Test 10: When bucket count exceeds 10,000, oldest-accessed buckets are evicted
    - Test 11: Thread safety — concurrent check() calls don't corrupt state (use threading.Lock)
  </behavior>
  <action>
    Create `jellyswipe/rate_limiter.py` implementing:

    1. **TokenBucket class** (pure data, no Flask dependency):
       - `__init__(self, capacity: int, refill_rate: float)` — `self.tokens = float(capacity)` per D-03, `self.last_refill = time.monotonic()`
       - `consume(self) -> bool` — refill elapsed tokens (`min(capacity, current + elapsed * refill_rate)`), then check if >= 1.0. If yes, subtract 1.0 and return True. Otherwise return False.
       - `retry_after(self) -> float` — if tokens >= 1.0 return 0.0, else `(1.0 - tokens) / refill_rate`
       - `last_access: float` — set to `time.monotonic()` on every `consume()` call (for eviction)

    2. **RateLimiter class**:
       - `__init__(self, max_buckets: int = 10000, stale_seconds: float = 300.0)` — `self._buckets: Dict[Tuple[str, str], TokenBucket] = {}`, `self._lock = threading.Lock()`
       - `check(self, endpoint: str, ip: str, limit: int, per_minutes: int = 1) -> Tuple[bool, float]`:
         - Acquire lock
         - Call `_evict_stale()` on each check (lazy eviction per D-22)
         - Key = `(endpoint, ip)` per D-05/D-07
         - If key not in dict, create new `TokenBucket(capacity=limit, refill_rate=limit / (per_minutes * 60))` per D-02
         - Call `bucket.consume()`
         - If refused: return `(False, bucket.retry_after())`
         - If allowed: return `(True, 0.0)`
       - `_evict_stale(self)` — remove entries where `time.monotonic() - bucket.last_access > stale_seconds` (D-21)
       - After eviction, if `len(self._buckets) > self.max_buckets`: sort by `last_access` ascending, remove oldest until under cap (D-23)
       - Use `threading.Lock` (gevent-aware via monkey-patching) — wrap the entire check method body in `with self._lock:`

    3. **Module-level singleton**: `rate_limiter = RateLimiter()` — Flask routes import this.

    Create `tests/test_rate_limiter.py` with the test cases listed in `<behavior>`. Use `time.sleep(0.05)` or mock `time.monotonic` for time-dependent tests. No Flask imports needed — these are pure Python unit tests.

    Per D-04: Zero new pip dependencies. Use only `time`, `threading`, `collections`, `typing` from stdlib.
