<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| pytest fixture setup -> temp database state | test bootstrap mutates per-test databases and runtime globals |
| test app fixtures -> dependency/provider overrides | auth/provider overrides can leak state across tests if teardown is incomplete |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-37-07 | Tampering | `tests/conftest.py` fixture bootstrap | mitigate | provision a fresh temp DB per test through `upgrade_to_head(...)` and set the runtime URL per fixture instance |
| T-37-08 | Repudiation | dependency override teardown | mitigate | keep explicit `dependency_overrides.clear()` and provider/rate-limiter reset in fixture teardown so test outcomes stay attributable |
| T-37-09 | Denial of Service | runtime reuse across tests | mitigate | dispose or reset the runtime between fixture instances instead of leaving one global engine pointed at the wrong temp DB |
</threat_model>
