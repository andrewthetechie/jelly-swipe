<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| request handler -> DB dependency | untrusted HTTP input reaches the per-request database transaction boundary |
| async runtime globals -> concurrent requests | engine/sessionmaker state is shared process-wide while sessions must remain per-request |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-37-01 | Elevation of Privilege | `jellyswipe/dependencies.py` | mitigate | yield a fresh `AsyncSession` per dependency call and never cache `AsyncSession` globally |
| T-37-02 | Tampering | `jellyswipe/routers/rooms.py` swipe transaction | mitigate | preserve `BEGIN IMMEDIATE` by executing the legacy SQL body through `DatabaseUnitOfWork.run_sync(...)` instead of rewriting the transaction semantics in this phase |
| T-37-03 | Denial of Service | `get_db_uow()` lifecycle | mitigate | rollback on exceptions and always close the session in `finally` so broken requests do not leak open transactions |
</threat_model>
