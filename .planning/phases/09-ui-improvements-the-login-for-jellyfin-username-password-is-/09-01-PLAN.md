# Plan 01 — Jellyfin server-delegated browser identity (no `prompt()` login)

<objective>
When `MEDIA_PROVIDER=jellyfin`, eliminate redundant browser username/password `prompt()` by binding the SPA to the **already-authenticated server provider** via a Flask session flag and extended `/auth/provider` metadata, without returning `JELLYFIN_API_KEY` or raw env secrets in any JSON response (`09-CONTEXT` D-01–D-04, `09-UI-SPEC` Interaction contract).
</objective>

<threat_model>
| ID | Threat | Severity | Mitigation |
|----|--------|----------|------------|
| T-09-01 | API key or server access token leaked in JSON to the browser | high | Never place `get_provider()` access token fields in `jsonify()` responses; only return `userId` string for UI header bootstrap |
| T-09-02 | Stale `localStorage` user token overrides delegate session | medium | On successful delegate POST, remove `provider_token` and `plex_token` keys (see `09-CONTEXT` D-04); document in SUMMARY |
| T-09-03 | Delegate route callable cross-origin to fixate session | medium | Keep same-site POST only; do not add permissive CORS; reject when `MEDIA_PROVIDER != "jellyfin"` with 400 |
</threat_model>

   - `server_primary_user_id_for_delegate(self) -> str`: calls `ensure_authenticated()`, returns the same user id string `_user_id()` already computes.
2. Both methods must raise the same style of `RuntimeError` messages already used for auth failures if credentials are unusable (do not add new external dependencies).
</action>

<action>
1. **Session key:** Use literal Flask session key `jf_delegate_server_identity` (bool `True` when active).
2. **`auth_provider`:** When `MEDIA_PROVIDER == "jellyfin"`, include JSON field `jellyfin_browser_auth` with string value `"delegate"` (always for this product — env creds are mandatory at boot).
3. **New route:** `POST /auth/jellyfin-use-server-identity` — if not jellyfin: `400` JSON `{"error": "Jellyfin delegate unavailable"}`. 4. **`_jellyfin_user_token_from_request()`:** If `session.get("jf_delegate_server_identity")` and `MEDIA_PROVIDER == "jellyfin"`, return `get_provider().server_access_token_for_delegate()` (catch wrong provider type with explicit guard).
5. **`_provider_user_id_from_request()`:** If delegate session active for jellyfin, return `get_provider().server_primary_user_id_for_delegate()` **before** reading `X-Jellyfin-User-Id` / token resolution, unless you intentionally document header override — default: delegate session wins to match single-household model.
6. Keep existing `/auth/jellyfin-login` route unchanged for emergency/compat (still no secret leakage on failure per Phase 3).
</action>
