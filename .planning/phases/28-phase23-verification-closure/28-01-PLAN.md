Two cleanup edits from the v1.6 milestone audit:

**Edit 1 — Remove dead `requests` import from jellyswipe/__init__.py:**
On line 14, the current import is:
```python
import sqlite3, os, random, re, requests, json, secrets, time
```
Remove `requests` from this import since all HTTP calls now go through `make_http_request()` (imported on line 15). The result should be:
```python
import sqlite3, os, random, re, json, secrets, time
```

NOTE: Verify that `requests` is NOT used anywhere else in `__init__.py` before removing. The only direct `requests` usage was in TMDB calls which are now all via `make_http_request()`. The `requests` import in `jellyfin_library.py` is still needed (for `requests.Session()` and `requests.RequestException`).

**Edit 2 — Remove redundant `raise_for_status()` from jellyfin_library.py:**
In the `server_info()` method, the fallback `except RuntimeError:` block uses `make_http_request()` which already calls `response.raise_for_status()` internally (http_client.py line 106). Remove the redundant `r.raise_for_status()` call (line 361).

The current code (lines 354-362):
```python
        except RuntimeError:
            response = make_http_request(
                method='GET',
                url=f"{self._base}/System/Info/Public",
                timeout=(5, 15)
            )
            r = response
            r.raise_for_status()
```

Should become:
```python
        except RuntimeError:
            response = make_http_request(
                method='GET',
                url=f"{self._base}/System/Info/Public",
                timeout=(5, 15)
            )
            pub = response.json()
```

Also update the variable references below: `pub = r.json()` is already there (line 362) but since we're renaming `r` to use `response` directly, change `pub = r.json()` to `pub = response.json()` as shown above. The rest of the method stays the same.

**After both edits, run the full test suite:**
```bash
python -m pytest tests/ -q
```
