<objective>
Migrate TMDB API authentication from v3 URL parameters to v4 Bearer token headers, eliminating credential exposure in logs and URLs.

Purpose: TMDB API keys in URL query strings are logged by every reverse proxy, visible in browser network tabs, and cached in intermediate systems. Bearer token authentication via headers eliminates this entire class of credential leakage.

Output: Code changes to `jellyswipe/__init__.py` (env var rename, URL cleanup, Bearer headers), updated test fixtures, and new TMDB auth verification tests.
</objective>
