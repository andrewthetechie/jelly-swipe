<task type="auto">
  <name>Task 2: Update pyproject.toml to include templates and static in package data</name>
  <files>pyproject.toml</files>
  <action>
    Update `pyproject.toml` to include `jellyswipe/templates/` and `jellyswipe/static/` in package data (per D-13).

    Using hatchling (the build backend from Phase 10), add the following configuration section to `pyproject.toml`:

    ```toml
    [tool.hatch.build.targets.wheel.shared-data]
    "jellyswipe/templates" = "jellyswipe/templates"
    "jellyswipe/static" = "jellyswipe/static"
    ```

    If there's already a `[tool.hatch.build]` section, add to it. If not, create it.

    This configuration tells hatchling to include the templates and static directories when building the wheel, so they're installed alongside the Python code.

    **Alternative approach:** If hatchling's default package discovery doesn't include these directories, also consider using:
    ```toml
    [tool.hatch.build]
    packages = ["jellyswipe"]

    [tool.hatch.build.targets.wheel]
    include = [
      "jellyswipe/**/*.py",
      "jellyswipe/templates/**/*",
      "jellyswipe/static/**/*",
    ]
    ```

    Use whichever approach works best with hatchling's configuration format. The goal is to ensure templates and static assets are included in the built package.
  </action>
  <verify>pyproject.toml includes package data configuration for templates and static</verify>
  <done>pyproject.toml configured to include templates and static in package data</done>
</task>
