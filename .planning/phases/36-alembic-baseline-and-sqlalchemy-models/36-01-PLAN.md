Implement four SQLAlchemy ORM classes:

- `Room` with `__tablename__ = "rooms"` and columns:
  - `pairing_code` primary key
  - `movie_data` `Text`, `nullable=False`, `server_default='[]'`
  - `ready` integer/bool-compatible column, `nullable=False`, `server_default='0'`
  - `current_genre` `Text`, `nullable=False`, `server_default='All'`
  - `solo_mode` integer/bool-compatible column, `nullable=False`, `server_default='0'`
  - `last_match_data`, `deck_position`, `deck_order` as `Text`, nullable

- `Swipe` with `__tablename__ = "swipes"` and columns:
  - `room_code` FK to `rooms.pairing_code` with `ondelete="CASCADE"`, `nullable=False`
  - `movie_id`, `user_id`, `direction` as non-null `Text`
  - `session_id` FK to `auth_sessions.session_id` with `ondelete="SET NULL"`, nullable
  - table indexes on `(room_code, movie_id, direction)` and `(room_code, movie_id, session_id)`

- `Match` with `__tablename__ = "matches"` and columns:
  - `room_code`, `movie_id`, `title`, `thumb`, `status`, `user_id` as non-null `Text`
  - `status` `server_default='active'`
  - `deep_link`, `rating`, `duration`, `year` as `Text`
  - unique constraint on `(room_code, movie_id, user_id)`
  - index on `(status, user_id)`
  - no DB foreign key from `room_code` to `rooms` because archived rows use `room_code='HISTORY'`

- `AuthSession` with `__tablename__ = "auth_sessions"` and columns:
  - `session_id` primary key
  - `jellyfin_token`, `jellyfin_user_id`, `created_at` as non-null `Text`
  - index on `created_at`

Add the full relationship graph:

- `Room.swipes`
- `Room.matches`
- `Swipe.room`
- `Swipe.auth_session`
- `AuthSession.swipes`
- `Match.room` using an explicit join condition that tolerates `room_code='HISTORY'`

Add `tests/test_models_metadata.py` to assert the metadata contains the four expected tables, the `auth_sessions` rename is present, and the `matches` table has no room FK while `swipes` does.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Alembic import path -> app package | importing the wrong module would trigger FastAPI env validation and provider startup |
| model graph -> existing raw SQL runtime | schema renames or constraints that are too aggressive will break the still-sync code paths before later phases convert them |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-36-01 | Denial of Service | Alembic metadata import | mitigate | keep a dedicated `jellyswipe.models.metadata` module that imports only models and `Base` |
| T-36-02 | Tampering | match archival history rows | mitigate | do not add a DB FK from `matches.room_code` to `rooms.pairing_code` while `"HISTORY"` remains a sentinel |
| T-36-03 | Elevation of Privilege | auth session FK cleanup | mitigate | use `ondelete="SET NULL"` on `swipes.session_id` so logout cleanup cannot delete swipe history |
</threat_model>
