<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| migration helper -> filesystem DB path | a bad URL builder or config path can migrate the wrong SQLite file |
| baseline DDL -> current sync runtime | if baseline constraints drift from existing raw SQL assumptions, app behavior will break before async conversion lands |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-36-04 | Tampering | Alembic revision truth | mitigate | hand-author the baseline from `db.py` and model definitions rather than trusting an unreviewed autogenerate pass |
| T-36-05 | Information Disclosure | Alembic importing app root | mitigate | keep `env.py` pointed only at `jellyswipe.models.metadata` |
| T-36-06 | Denial of Service | wrong SQLite path migrated | mitigate | centralize URL creation in `jellyswipe/migrations.py` and cover it with focused tests |
</threat_model>
