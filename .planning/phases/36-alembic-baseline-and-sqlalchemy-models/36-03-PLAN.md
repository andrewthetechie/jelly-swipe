<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| runtime connection helper -> SQLite FK enforcement | foreign keys exist in DDL only if every connection enables them |
| test bootstrap -> migrated temp DB | stale fixture code could silently recreate a second schema path and undermine the migration rollout |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-36-07 | Tampering | dual bootstrap paths | mitigate | remove `init_db()` schema creation and update fixtures to use `upgrade_to_head(...)` exclusively |
| T-36-08 | Integrity | SQLite FK enforcement | mitigate | set `PRAGMA foreign_keys=ON` on every connection via `configure_sqlite_connection()` |
| T-36-09 | Denial of Service | startup on missing schema | accept | Phase 36 intentionally leaves manual/bootstrap migration explicit; Phase 37 wires the startup wrapper before Uvicorn |
</threat_model>
