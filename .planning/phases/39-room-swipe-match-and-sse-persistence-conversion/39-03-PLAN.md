<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| authenticated request -> swipe service | User-controlled movie IDs, directions, and session-backed participant identity reach the mutation layer |
| swipe service -> SQLite transaction | Critical multi-statement mutation must serialize correctly under concurrent access |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-39-07 | T | serialized swipe mutation | mitigate | Keep explicit `BEGIN IMMEDIATE` in the sync bridge and keep the bridge under one dependency-owned transaction to preserve SQLite write serialization. |
| T-39-08 | R | undo/delete recomputation | mitigate | Recompute `rooms.last_match_data` from persisted active matches after every undo/delete so visible state always reflects committed rows. |
| T-39-09 | E | participant identity separation | mitigate | Use session-backed `session_id` when present and never accept client-supplied identity fields for match participation. |
</threat_model>
