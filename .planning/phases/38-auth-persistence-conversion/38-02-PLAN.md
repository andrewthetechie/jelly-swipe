## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser cookie jar -> auth routes | untrusted client cookie state drives login-required behavior here |
| auth router -> auth dependency/service | route handlers trust the dependency/service seam to authorize and mutate local session state correctly |
| `/me` route -> room persistence check | auth route reads room state through an unconverted sync room path that must remain behaviorally compatible until Phase 39 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-38-05 | Spoofing | `/me`, `/auth/logout`, protected auth routes | mitigate | keep `Depends(require_auth)` as the only auth gate and preserve the exact 401 contract on missing persisted sessions |
| T-38-06 | Tampering | stale cookie/session state | mitigate | route tests must prove stale `session_id` misses clear the client session cookie/state, not just raise 401 |
| T-38-07 | Repudiation | logout success despite best-effort delete failure | mitigate | route output remains `{"status": "logged_out"}` while Plan 01 logs `auth_session_delete_failed`; route tests confirm visible behavior stays correct |
| T-38-08 | Elevation of Privilege | auth route accidentally reaching room persistence migration scope | mitigate | keep `/me` room lookup on the existing sync room helper and avoid introducing room/swipe/match/SSE repository work in this phase |
</threat_model>
