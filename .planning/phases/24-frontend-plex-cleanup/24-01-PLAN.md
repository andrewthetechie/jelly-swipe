**2. Add new route in `jellyswipe/__init__.py` (per D-03):**

Add a new route AFTER the existing `/auth/jellyfin-login` route (after line 272). Pattern follows existing routes:

```python
@app.route("/jellyfin/server-info", methods=["GET"])
def jellyfin_server_info():
    try:
        info = get_provider().server_info()
        return jsonify({"baseUrl": info.get("machineIdentifier", ""), "webUrl": info.get("webUrl", "")})
    except Exception:
        return jsonify({"baseUrl": "", "webUrl": ""}), 200
```

Key design decisions:
- Returns 200 even on error (frontend handles gracefully per D-07)
- Only exposes `baseUrl` (machineIdentifier) and `webUrl` — no credentials, tokens, or server secrets
- Uses the existing `server_info()` method which already handles auth fallback via `/System/Info/Public`
