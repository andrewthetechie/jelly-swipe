```python
@app.after_request
def add_csp_header(response):
    """Add Content Security Policy header to block inline scripts and restrict external resources."""
    csp_policy = (
        "default-src 'self'; "
        "script-src 'self'; "
        "object-src 'none'; "
        "img-src 'self' https://image.tmdb.org; "
        "frame-src https://www.youtube.com"
    )
    response.headers['Content-Security-Policy'] = csp_policy
    return response
```

| Flask app → Jellyfin API | Trusted metadata source (mocked in tests) |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-22-01 | Tampering | test_swipe_ignores_client_supplied_title_thumb | accept | Test validates security behavior, not production code. No mitigation needed for test itself. |
| T-22-02 | Spoofing | test_csp_header_present_on_responses | accept | Test verifies header presence, not production behavior. No mitigation needed for test itself. |
| T-22-03 | Info disclosure | test logs (caplog) | accept | Test logs contain simulated attack data for validation purposes only. No real PII or secrets. |
| T-22-04 | Elevation of privilege | Mocked session/user_id | accept | Test uses mock session to simulate authenticated user. No real authentication bypass. |

**Note:** This is a testing phase, not production code. The threat model focuses on ensuring tests accurately validate security behavior without introducing new vulnerabilities.
