<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SSE session state → stream generator | SSE generator reads request.session["active_room"] — if cookie is malformed, generator sees empty dict and produces room-not-found events instead of stream events |
| test SSE buffering assumption | TestClient buffers the full SSE response body — if the generator does not terminate (deadline mocking fails), test hangs indefinitely |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-35-10 | Denial of Service | SSE test hang if deadline mocking fails | mitigate | Keep monkeypatch.setattr(time, "time", ...) deadline mocking — this is the actual timing control mechanism for FastAPI asyncio.sleep-based SSE route |
| T-35-11 | Spoofing | _gevent_sleep monkeypatch raising=False | accept | Adding raising=False is strictly defensive — the attribute does not exist in the FastAPI app so this is a no-op; no security impact |
| T-35-12 | Tampering | SSE event format verification via response.text | mitigate | response.text is the str-decoded full buffered SSE body; string assertions on event format ('"closed": true', 'data:') are direct equivalents of the previous response.data.decode() assertions |
</threat_model>
