**set_session_cookie helper** (add as a module-level function, before fixtures):
```python
def set_session_cookie(client, data: dict, secret_key: str) -> None:
    """Inject session state into a FastAPI TestClient's cookie jar.

    Replicates Starlette 1.0.0 SessionMiddleware signing format exactly:
      base64(json_bytes).timestamp.signature via itsdangerous.TimestampSigner.

    VERIFIED against starlette.middleware.sessions source code.
    Do NOT use URLSafeTimedSerializer — that is Flask's format, not Starlette's.
    """
    signer = itsdangerous.TimestampSigner(str(secret_key))
    payload = b64encode(json.dumps(data).encode("utf-8"))
    signed = signer.sign(payload)
    client.cookies.set("session", signed.decode("utf-8"))
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| test helper → Starlette session | set_session_cookie crafts a signed cookie; if the signer variant is wrong, Starlette silently ignores it and session reads empty dict — could mask test coverage |
| fixture teardown → test isolation | dependency_overrides not cleared between tests allows auth bypass state to leak across test functions, producing false-positive auth tests |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-35-01 | Spoofing | set_session_cookie helper | mitigate | Use `itsdangerous.TimestampSigner` (verified Starlette 1.0.0 format) — NOT URLSafeTimedSerializer; signing key must match test_config SECRET_KEY exactly |
| T-35-02 | Elevation of Privilege | dependency_overrides leakage between tests | mitigate | `fast_app.dependency_overrides.clear()` placed AFTER yield in both `app` and `app_real_auth` fixtures — enforced by yield/teardown pattern (ASVS V3.1 session isolation) |
| T-35-03 | Elevation of Privilege | app_real_auth fixture accidentally set on wrong tests | accept | Fixture naming is explicit (`client_real_auth` vs `client`); test files must opt in deliberately; risk is low in a test-only context |
</threat_model>
