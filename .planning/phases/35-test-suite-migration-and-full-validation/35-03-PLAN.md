## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| test vault seeding → real require_auth | test_route_authorization.py seeds user_tokens with a session_id; if the cookie's session_id doesn't match the vault, require_auth raises 401 — session cookie signing must be exact |
| session_transaction read-back replacement | Replacing session read-back with API call changes WHAT is verified — must ensure the follow-up endpoint actually exercises the session state that was set |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-35-07 | Spoofing | session cookie with session_id for real-auth tests | mitigate | set_session_cookie uses TimestampSigner (verified Starlette 1.0.0 format); session_id in cookie must exactly match value inserted into user_tokens table |
| T-35-08 | Elevation of Privilege | test_route_authorization.py accidentally using override fixture | mitigate | Local client fixture removed; tests declare `client_real_auth` explicitly — cannot accidentally receive the override client (D-02 enforced structurally) |
| T-35-09 | Repudiation | session read-back verification weakened | mitigate | Follow-up API call to `/auth/provider` (200 = session is valid) is equivalent or stronger proof than reading sess["session_id"] directly — auth failure would produce 401 |
</threat_model>
