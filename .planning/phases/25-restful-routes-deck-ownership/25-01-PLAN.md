## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → API routes | Untrusted input crosses here — room code in URL, movie_id/direction in body |
| SSE stream → client | Unauthenticated endpoint — room code in URL is sole access control |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-25-01 | Spoofing | `/room/<code>/stream` (no auth) | mitigate | Room codes are 4-digit random (1000-9999). Brute-force feasible but low-value target (read-only SSE with match data only). Acceptable risk for MVP. |
| T-25-02 | Tampering | `/room/<code>/swipe` body | accept | `@login_required` ensures authenticated users only. Extra fields in body (title, thumb) are silently ignored per D-07. |
| T-25-03 | Info disclosure | `/room/<code>/deck` | mitigate | `@login_required` protects endpoint. User can only see decks for rooms they know the code for. |
| T-25-04 | Elevation of privilege | `/room/<code>/join` | mitigate | `@login_required` required. Room code is not secret but requires knowledge of pairing code. |
</threat_model>
