    ```python
    # After the regular swipe INSERT (which uses autocommit via context manager),
    # open a separate IMMEDIATE transaction for the match check:
    conn.execute('BEGIN IMMEDIATE')
    try:
        # ... match check and insert logic ...
        conn.execute('COMMIT')
    except Exception:
        conn.execute('ROLLBACK')
        raise
    ```
    Note: The swipe record INSERT and cursor advance should remain OUTSIDE the IMMEDIATE transaction (they don't need it). Only the match check-then-insert pair needs IMMEDIATE isolation.

    ```python
    rows = conn.execute('SELECT title, thumb, movie_id, deep_link, rating, duration, year FROM matches WHERE ...')
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → POST /room/{code}/swipe | Untrusted input: movie_id, direction — direction validated, movie_id used in DB queries |
| swipe → SSE stream | Internal: match data written to rooms table, read by SSE generator |
| server → Jellyfin | Server-side only: deep link URL constructed from trusted env var JELLYFIN_URL |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-26-01 | S | POST /room/{code}/swipe | mitigate | @login_required ensures authenticated users only; g.user_id from vault, not client-supplied |
| T-26-02 | T | matches table INSERT | mitigate | INSERT OR IGNORE prevents duplicate match injection; BEGIN IMMEDIATE prevents TOCTOU race |
| T-26-03 | I | GET /matches | mitigate | @login_required + user_id filter ensures users only see their own matches |
| T-26-04 | D | BEGIN IMMEDIATE | accept | Slight performance impact from SQLite locking; acceptable for single-server self-hosted deployment |
| T-26-05 | E | movie_id in deep link | mitigate | movie_id is from Jellyfin item ID (UUID format); validated by regex in proxy route pattern |
</threat_model>
