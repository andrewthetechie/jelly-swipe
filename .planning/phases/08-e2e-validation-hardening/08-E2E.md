# Jellyfin-forward operator E2E (milestone narrative)

Doc-first, operator-run steps for **`MEDIA_PROVIDER=jellyfin`**: login → room → deck → swipe/match → trailer and cast. Fill **Date / Environment / Result** as you execute; do not paste real API keys, user tokens, or `Authorization:` header values (use placeholders such as `***REDACTED***`).

## Preconditions

| Date | Environment | Result |
|------|-------------|--------|
| | | |

Set these **names** only (values stay in your local `.env` or shell, not in this file):

- `MEDIA_PROVIDER=jellyfin`
- `JELLYFIN_URL` — base URL of your Jellyfin server
- **Auth (one path):** either `JELLYFIN_API_KEY`, or `JELLYFIN_USERNAME` + `JELLYFIN_PASSWORD` (see Phase 3 verification for the contract)
- `TMDB_API_KEY` — TMDB key for trailer/cast routes
- `FLASK_SECRET` — Flask session secret

Optional: `DB_PATH` if you are not using the default SQLite path.

## Jellyfin login and session

| Date | Environment | Result |
|------|-------------|--------|

1. Start the Flask app with Jellyfin env vars loaded (see Phase 3 [03-VERIFICATION.md](../03-jellyfin-authentication-http-client/03-VERIFICATION.md) for import smoke patterns).
2. In the browser, open `/` and complete Jellyfin login per the UI (client uses `/auth/jellyfin-login` and related routes in `app.py`).
3. Confirm the session carries a **MediaBrowser**-style token in client requests (describe behavior only; do not paste the token).

Evidence and threat notes: [07-VERIFICATION.md](../07-verification-closure-jellyfin-parity/07-VERIFICATION.md), [03-VERIFICATION.md](../03-jellyfin-authentication-http-client/03-VERIFICATION.md).

## Room create and deck load

| Date | Environment | Result |
|------|-------------|--------|

1. Create a room via `POST /room/create` (or the UI flow that calls it) with Jellyfin-backed auth headers as required by your deployment.
2. Load the deck: `GET /movies` / `GET /genres` as applicable; confirm HTTP status and that returned card objects include an **`id`** field suitable for downstream routes (Jellyfin item id used by the provider — see [04-VERIFICATION.md](../04-jellyfin-library-media/04-VERIFICATION.md) and `media_provider/jellyfin_library.py` `fetch_deck` mapping).

## Swipe / match flow

| Date | Environment | Result |
|------|-------------|--------|

1. `POST /room/swipe` with `movie_id` and direction from the active deck card’s `id`.
2. When both sides agree, confirm `GET /matches` (or UI) shows the expected match row.

Cross-check user-isolation expectations: [05-VERIFICATION.md](../05-user-parity-packaging/05-VERIFICATION.md).

## Trailer and cast

| Date | Environment | Result |
|------|-------------|--------|

1. Pick a `movie_id` from an **active Jellyfin-backed deck** JSON field named **`id`** (same id string the UI uses for swipes).
2. `GET /get-trailer/<movie_id>` — expect JSON with a YouTube trailer key or a documented 404 path from TMDB/Jellyfin resolution (`app.py` route `get_trailer`).
3. `GET /cast/<movie_id>` — expect cast list JSON (`app.py` route `get_cast`).

Field parity evidence: [04-VERIFICATION.md](../04-jellyfin-library-media/04-VERIFICATION.md).

## Dual browser / identity isolation (JUSR-01 spirit)

| Date | Environment | Result |
|------|-------------|--------|

Run two separate browser profiles (or one normal + one private window) logged in as **different** Jellyfin identities. Exercise the same room code where applicable and confirm favorites / swipe attribution do not bleed across users (see JUSR rows in [05-VERIFICATION.md](../05-user-parity-packaging/05-VERIFICATION.md)).

## Non-secret startup / import check

Mirrors Phase 3 style: clean environment, dummy secrets, no listener required on the Jellyfin port (import only).

```bash
env -i PATH="$PATH" HOME="$HOME" \
  MEDIA_PROVIDER=jellyfin \
  JELLYFIN_URL=http://127.0.0.1:8096 \
  FLASK_SECRET=dummy TMDB_API_KEY=dummy \
  python -c "from app import app; print('import_ok', app.name)"
```
